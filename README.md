# Documentation

This directory contains the documentation for the repository.

## Overview

- Purpose: Provide user-facing documentation, guides, and developer notes.
- Location: docs/

## How to contribute

1. Edit or add markdown files in the `docs/` directory.
2. Create a branch for your change: `git checkout -b docs/update-example`
3. Commit your changes with a clear message.
4. Open a pull request describing the changes.

## Recommended files

- `docs/README.md` (this file)
- `docs/usage.md` — usage examples and quick start
- `docs/development.md` — developer setup and contributing guidelines
